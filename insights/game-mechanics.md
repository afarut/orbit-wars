# Механики Orbit Wars — что важно для бота

**Дата:** 2026-06-14 · **Источник:** официальный спек Kaggle (полная версия — `orbit_wars_rules.md` в корне) · **Зачем:** выжимка нетривиальных правил, влияющих на стратегию и на разметку фич/таргетов.

> Это не пересказ всех правил (они в `orbit_wars_rules.md`), а список механик, которые **меняют решения агента** и которые легко проглядеть при обучении.

---

## Цель и скоринг

- Поле **100×100**, непрерывное, начало координат — верхний-левый угол. Солнце в **(50, 50)**, радиус **10**.
- Игра длится **500 ходов** (или до элиминации — остаётся ≤1 игрок с планетами/флотами).
- **Победа = больше всего кораблей** (на планетах + во флотах) в конце. **Не территория, а суммарный счёт.**
  - Следствие: накопление гарнизона на богатых production-планетах — валидная стратегия, не обязательно захватывать всё.
  - Margin победы на рейтинг **не влияет** (только win/loss/draw) — но локально для SFT мы учимся именно у победителя (см. [[dataset-format]]).

## Карта и симметрия

- Все планеты/кометы расставлены с **4-кратной зеркальной симметрией** вокруг центра: `(x,y), (100-x,y), (x,100-y), (100-x,100-y)`.
  - Можно использовать симметрию как инвариант для аугментации данных (поворот/отражение обзора = валидная игра).
- **20–40 планет** (5–10 групп по 4). Минимум 3 группы статичные, минимум 1 орбитальная.
- Домашние планеты — одна случайная группа, старт **10 кораблей**. 2p: диагонально (Q1/Q4); 4p: по одной из группы.

## Планеты

`[id, owner, x, y, radius, ships, production]`, `owner=-1` — нейтрал.

- **production 1–5** кораблей/ход, **radius = 1 + ln(production)** → крупные планеты ценнее И физически больше (легче в них попасть/быть сметённым).
- Стартовый гарнизон **5–99**, скошен в низ.
- **Орбитальные** (`orbital_radius + planet_radius < 50`) вращаются вокруг солнца с постоянной `angular_velocity` (**0.025–0.05 рад/ход**, фикс на игру). Позиции предсказуемы через `initial_planets` + `angular_velocity` → **прицеливаться надо в будущую точку, а не текущую.**
- **Статичные** (дальше от центра) не вращаются.

## Флоты — ключевые тонкости

`[id, owner, x, y, angle, from_planet_id, ships]`. Действие: `[from_id, angle, ships]`.

- **Скорость зависит от размера** (лог-кривая):
  `speed = 1 + (maxSpeed-1) * (ln(ships)/ln(1000))^1.5`, maxSpeed=6.
  1 корабль → 1.0; ~500 → ~5; ~1000 → max. **Дробить флот = терять скорость**; большой флот доезжает быстрее.
- **Три способа потерять флот целиком:**
  1. вышел за поле (100×100),
  2. **пересёк солнце** (отрезок пути зашёл в радиус 10) — вокруг центра надо облетать,
  3. столкнулся с любой планетой (отрезок пути в радиусе планеты) → **бой**.
- **Continuous collision** — проверяется весь отрезок старая→новая позиция, не только конец. Нельзя «проскочить» сквозь планету за один тик.
- Нельзя отправить больше кораблей, чем сейчас в гарнизоне (после применения сдвига кадра в датасете это держится — см. [[sft-shipcount-analysis]]).
- Флот спавнится **сразу за радиусом** планеты в заданном направлении; можно несколько вылетов за ход с одной/разных планет.

## Кометы

- Спавнятся группами по 4 (по одной на квадрант) на шагах **50, 150, 250, 350, 450**. Радиус 1.0, production 1/ход.
- Стартовые корабли — min из 4 бросков 1–99 (скошены вниз), общие на всю группу.
- Скорость `cometSpeed=4.0`. В obs: `comet_planet_ids` + `comets{paths, path_index}` → **траектория известна наперёд**, позиции предсказуемы.
- **Кометы удаляются ДО вылетов** в начале хода вместе с гарнизоном → с уходящей кометы вылететь нельзя; держать на ней корабли рискованно у края поля.
- Во всём остальном комета = обычная планета (захват/production/бой).

## Порядок хода (важен для предсказаний)

1. удаление улетевших комет → 2. спавн комет → 3. **вылеты (действия игроков)** → 4. production у всех владельцев → 5. движение флотов (проверка границ/солнца/планет, бои в очередь) → 6. вращение планет и движение комет (**движущаяся планета может сама смести флот в бой**) → 7. разрешение боёв.

- Тонкость для прицеливания: production начисляется (шаг 4) **после** вылета (шаг 3) того же хода, а планеты двигаются (шаг 6) **после** движения флотов (шаг 5).

## Бой

При столкновении флотов с планетой:

1. флоты группируются по владельцу, корабли одного владельца суммируются;
2. **крупнейшая атакующая сила бьётся со второй по величине → выживает разница**;
3. если выживший атакующий:
   - тот же владелец, что и планета → добавляется в гарнизон;
   - другой владелец → бьётся с гарнизоном; если **превысил** гарнизон — планета меняет владельца, гарнизон = излишек;
4. **ничья двух атакующих → уничтожаются все атакующие** (0 выживших).

- Следствие: для захвата нужно `> ships` гарнизона (ровно равно — взаимоуничтожение, планета не берётся). Бейзлайн-снайпер в `orbit_wars_rules.md` шлёт `ships+1`.

## Бюджет времени

- `actTimeout=1` сек/ход, есть `remainingOverageTime` — общий запас на эпизод. Инференс модели должен укладываться, иначе ход теряется.
